# Review Report: AI Lab Investment

**Reviewer:** Codex
**Date:** 2026-03-13

## Executive Summary
The repository is well organized, the core model modules are readable, and the baseline single-firm and unleveraged duopoly numbers in the manuscript reproduce closely from the current code. The mathematical backbone for Proposition 1 and the baseline regime-switching setup is mostly implemented consistently, and the test suite is broad on the model layer: `190` tests pass. The main weakness is not that the project lacks structure, but that several headline quantitative claims in the paper are not generated by the code path the paper actually uses.

The most important problems are in the credit-risk and valuation sections. The paper states first-passage default probabilities and Dario's-dilemma tail-risk numbers that are not produced consistently by the current implementation. In particular, `dario_dilemma_leveraged()` uses a simpler `N(-d2)` approximation instead of the exact first-passage formula used elsewhere, and the published credit-risk figure is generated with conventions that contradict the text ("evaluated at `3 * X_D`" and low default probabilities). The default `just run-pipeline` command also does not reproduce substantive outputs because every task flag is disabled in `conf/config.yaml`. I would assess the codebase as promising but not submission-ready, and the paper as requiring major revision before submission.

## Part 1: Code Validation
### 1. Mathematical Correctness

#### Propositions vs. code
- **Pass:** Proposition 1's main objects match the implementation. The paper's effective revenue coefficient `A_eff` in [`paper/_model.qmd`](../paper/_model.qmd) lines 76-81 matches [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 431-454. The trigger formula in [`paper/_model.qmd`](../paper/_model.qmd) lines 180-183 matches [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 538-541. The closed-form `K*` in [`paper/_model.qmd`](../paper/_model.qmd) lines 128-141 numerically matches the optimizer output.
- **Pass with qualification:** Proposition 2's default-boundary formula in [`paper/_model.qmd`](../paper/_model.qmd) lines 255-315 matches [`src/ai_lab_investment/models/duopoly.py`](../src/ai_lab_investment/models/duopoly.py) lines 303-351 and 432-467. The "faith-based survival" channel is represented through `A_eff` in the denominator.
- **Has issues:** Proposition 3's appendix claims computational verification of single crossing on a 500-point grid and equilibrium uniqueness from 10 random starts, but the implementation does not perform those checks. The appendix states this in [`paper/_appendix.qmd`](../paper/_appendix.qmd) lines 229-235, while [`src/ai_lab_investment/models/duopoly.py`](../src/ai_lab_investment/models/duopoly.py) lines 815-836 only run one Brent root solve on a fixed bracket and the optimizers at lines 592-603 and 725-735 use deterministic starting grids, not 10 random starts.

#### Proofs
- **Pass:** Proposition 1's proof in [`paper/_appendix.qmd`](../paper/_appendix.qmd) lines 39-105 is logically coherent. The separability argument is exactly the structure used in code: `A_eff = g(phi) * K^alpha`, and the optimizer objective in [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 474-495 reflects that factorization.
- **Pass with qualification:** Proposition 2(ii)'s two-channel derivative discussion in [`paper/_appendix.qmd`](../paper/_appendix.qmd) lines 119-149 is plausible and numerically consistent with the implemented formula, but the dominance claim is empirical, not proved. The appendix presents it appropriately as such.
- **Has issues:** The Dario's-dilemma Taylor argument is only heuristic, which is acceptable if described as heuristic, but the paper occasionally writes as if the sign result is established rather than numerically observed. See [`paper/_appendix.qmd`](../paper/_appendix.qmd) lines 192-210 and [`paper/_valuation.qmd`](../paper/_valuation.qmd) lines 103-123.

#### Two model modes
- **Pass:** The simple mode and full mode are distinct in code and consistent with the top-level instructions. Simple mode is in [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 47-325; full `phi` mode is in lines 431-567.
- **Has issues:** The paper says all reported paper figures use the full mode, but the growth decomposition figure is described as an H-regime decomposition around `K_H^*` while the code actually uses `installed_value_with_phi(..., "L")` and the full L-regime trigger. See [`paper/_valuation.qmd`](../paper/_valuation.qmd) lines 35-42 versus [`src/ai_lab_investment/figures/paper.py`](../src/ai_lab_investment/figures/paper.py) lines 532-548.

#### Numerical methods
- **Pass:** The single-firm and duopoly optimizers do use multiple starts and bounded/bracketed methods. See [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 515-524 and [`src/ai_lab_investment/models/duopoly.py`](../src/ai_lab_investment/models/duopoly.py) lines 592-603 and 725-735.
- **Has issues:** The code never checks `result.success` from Nelder-Mead, only whether `best_val` improved. A failed optimizer can still be accepted if it returns a finite objective. This affects [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 519-527 and [`src/ai_lab_investment/models/duopoly.py`](../src/ai_lab_investment/models/duopoly.py) lines 597-606 and 730-738.
- **Pass:** Brent root-finding is used correctly where a sign change exists.
- **Has issues:** The appendix claims 500-point single-crossing checks and 10 random starts for equilibrium uniqueness, but those are not implemented.

#### Parameter consistency
- **Pass:** Baseline parameter values in [`src/ai_lab_investment/models/parameters.py`](../src/ai_lab_investment/models/parameters.py) lines 41-64 match the paper's baseline in [`paper/_appendix.qmd`](../paper/_appendix.qmd) lines 243-260 and [`paper/_calibration.qmd`](../paper/_calibration.qmd) lines 14-40.
- **Pass:** The baseline outputs reproduce well: `X* = 0.0047218`, `K* = 0.0067266`, `phi* = 0.7009`, `X_F = 0.5743`, `K_F = 1.3021`, `X_P = 0.00818`, `X_H^mono = 0.01627`, all close to the manuscript.
- **Could not verify fully:** External data sources and citation accuracy were not independently checked against the underlying public filings in this local-only review.

#### Regime switching
- **Pass:** The GBM with absorbing H regime in [`paper/_model.qmd`](../paper/_model.qmd) lines 7-21 is implemented correctly in [`src/ai_lab_investment/models/base_model.py`](../src/ai_lab_investment/models/base_model.py) lines 327-372. The switch only occurs from `L` to `H`, and `H` remains absorbing.

#### Default probability
- **Has issues:** The standalone `default_probability()` implementation uses the exact first-passage formula and matches the equation in [`paper/_valuation.qmd`](../paper/_valuation.qmd) lines 63-68. See [`src/ai_lab_investment/models/valuation.py`](../src/ai_lab_investment/models/valuation.py) lines 149-213.
- **Critical issue:** `dario_dilemma_leveraged()` does **not** use that exact formula. Its internal `_default_prob()` in [`src/ai_lab_investment/models/valuation.py`](../src/ai_lab_investment/models/valuation.py) lines 399-412 uses only `N(-d2)`, not the first-passage expression. This materially changes the reported tail-risk numbers.

### 2. Code Quality and Testing

#### Test meaningfulness
- **Pass with qualification:** The tests in [`tests/test_base_model.py`](../tests/test_base_model.py), [`tests/test_duopoly.py`](../tests/test_duopoly.py), [`tests/test_symbolic_duopoly.py`](../tests/test_symbolic_duopoly.py), and [`tests/test_valuation.py`](../tests/test_valuation.py) do check real economic properties: monotonicity in leverage, sign of derivatives, no-trigger regions, contest-share behavior, and positivity of option values.
- **Has issues:** The tests rarely check the actual headline numbers quoted in the paper. For example, [`tests/test_valuation.py`](../tests/test_valuation.py) lines 72-114 only verify broad spread/probability monotonicity and array shapes, which is why the credit-risk figure/text mismatch is not caught.

#### Edge cases
- **Pass:** There is good coverage of some boundaries: no leverage, `lambda=0`, cases without L triggers, and endogenous lambda special cases.
- **Has issues:** There is no test that the figure conventions match the paper's stated evaluation points, no test for the exact Dario leveraged default-probability formula, and no test that `just run-pipeline` produces substantive outputs.

#### Numerical stability
- **Pass with qualification:** The code has guards against invalid logs, zero denominators, and infeasible `phi`/leverage regions.
- **Has issues:** Silent acceptance of optimizer failures remains a risk because `result.success` is ignored.

#### Code organization
- **Pass:** The core economic logic is separated cleanly across parameters, single-firm model, duopoly model, valuation, and figures. The repository structure is good.
- **Has issues:** The valuation layer mixes paper-facing quantities and convenience summaries without enough invariants. For example, `growth_option_decomposition()` in [`src/ai_lab_investment/models/valuation.py`](../src/ai_lab_investment/models/valuation.py) lines 67-82 double-counts regime-switch value in regime `L`: `option_val` already includes switching, then `regime_switch` is added again to `total`.

#### Reproducibility
- **Has issues:** `just run-pipeline` succeeds, but only because the default Hydra config disables every task. See [`conf/config.yaml`](../conf/config.yaml) lines 9-23 and [`src/ai_lab_investment/pipeline.py`](../src/ai_lab_investment/pipeline.py) lines 124-130. This does not reproduce results; it is effectively a no-op.
- **Pass:** `uv run python paper/generate_figures.py` completes successfully and regenerates the 11 paper figures.

## Part 2: Paper Review
### 3. Paper Content Review

#### 3a. Structure and Argument
- **Motivation: Pass.** The introduction in [`paper/_introduction.qmd`](../paper/_introduction.qmd) lines 1-34 is strong and clearly frames the timing, scale, and allocation problem.
- **Literature positioning: Pass.** The literature review in [`paper/_literature.qmd`](../paper/_literature.qmd) lines 1-26 is broad and well targeted. RFS/JF readers will recognize the positioning immediately.
- **Model building: Pass.** The progression from single firm to duopoly is natural, and the duopoly justification in [`paper/_introduction.qmd`](../paper/_introduction.qmd) lines 31-35 is credible.
- **Key assumptions: Pass with qualification.** Assumptions A1-A4 are clearly stated in [`paper/_model.qmd`](../paper/_model.qmd) lines 115-126. A3 is explained well. The main limitation is that several later quantitative claims rely on conventions that are not tightly aligned with those assumptions in code.
- **Conclusion: Has issues.** The conclusion contradicts the paper's own main numerical finding. [`paper/_conclusion.qmd`](../paper/_conclusion.qmd) lines 16-19 say "the downside of overinvestment ... exceeds the opportunity cost of underinvestment," which is the opposite of [`paper/_valuation.qmd`](../paper/_valuation.qmd) lines 103-123 and [`paper/_appendix.qmd`](../paper/_appendix.qmd) lines 196-210, where underinvestment is costlier in expected value.

#### 3b. Writing Quality
- **Clarity: Pass with issues.** Most sections are clear. The main clarity problem is not prose quality but inconsistent quantitative narration across sections.
- **Notation: Pass.** Notation is mostly consistent and definitions usually precede use.
- **Length and focus: Pass with qualification.** The paper is ambitious but still within top-journal scope. Some appendix claims overstate what is analytically versus computationally established.
- **Abstract: Could not verify.** No abstract text was reviewed in this pass.

#### 3c. Journal Fit
- **Contribution significance: Pass with qualification.** The topic is timely and the mechanism is interesting, but the current quantitative inconsistencies lower credibility materially.
- **Methodological rigor: Has issues.** The model is sophisticated, but the paper currently overstates computational verification in places and contains contradictory interpretations of the main asymmetry result.
- **Formatting and conventions: Pass.** The manuscript broadly follows formal finance/econ style.
- **Which journal fits best:** `RFS` is the best fit. The paper is a corporate finance / investment-under-competition model with a timely application and a strong strategic-finance angle. It is less theorem-heavy than a typical `Econometrica` submission and more mechanism/calibration driven than a typical `JF` paper.

### 5. Calibration and Results

#### Parameter values
- **Pass with qualification:** The baseline primitive parameters are internally consistent between code and paper.
- **Could not verify fully:** The external source reasonableness of each firm-level calibration input was not independently checked against the cited documents here.

#### Sensitivity
- **Pass with qualification:** The paper discusses sensitivity and the code has comparative-statics routines.
- **Has issues:** The appendix claims more extensive numerical verification than is concretely implemented.

#### Comparative statics
- **Pass:** The reproduced comparative statics are directionally sensible: higher `lambda` raises `phi*` and lowers the trigger; higher leverage raises the default boundary; preemption lowers the leader trigger.

#### Dario's dilemma results
- **Critical issue:** The paper's expected-value asymmetry is broadly consistent with `dario_dilemma()`, but the leveraged tail-risk numbers depend on formula choice. The code path used inside `dario_dilemma_leveraged()` is not the exact first-passage formula stated in the paper.
- **Evidence:** Using the exact first-passage helper `default_probability(..., regime="L")` at the entry points gives approximately `0.64%` baseline, `0.79%` conservative, and `5.04%` aggressive, which matches the manuscript. The current `dario_dilemma_leveraged()` instead reports about `0.34%`, `0.42%`, and `2.69%`.

#### Growth decomposition
- **Has issues:** The current growth decomposition is not internally consistent with the paper's interpretation. The figure and text present it as a decomposition around optimal capacity, but the code's "capacity gap value" goes to zero before `K/K* = 1`.

## Summary of Issues
### Critical Issues
- Credit-risk and tail-risk calculations are inconsistent across code and paper. The exact first-passage formula in [`src/ai_lab_investment/models/valuation.py`](../src/ai_lab_investment/models/valuation.py) lines 149-213 is not used inside `dario_dilemma_leveraged()` at lines 399-412, even though the paper presents exact first-passage default probabilities.
- The paper's credit-risk text and caption do not match the generated figure. See [`paper/_valuation.qmd`](../paper/_valuation.qmd) lines 71-77 versus [`src/ai_lab_investment/models/valuation.py`](../src/ai_lab_investment/models/valuation.py) lines 135-137 and 233-244 and [`paper/figures/fig_credit_risk.png`](../paper/figures/fig_credit_risk.png).
- The conclusion reverses the sign of the paper's main expected-value asymmetry. See [`paper/_conclusion.qmd`](../paper/_conclusion.qmd) lines 16-19 versus [`paper/_valuation.qmd`](../paper/_valuation.qmd) lines 103-123.

### Major Issues
- `just run-pipeline` is not a meaningful reproducibility check because the default config disables all tasks. See [`conf/config.yaml`](../conf/config.yaml) lines 9-23.
- The growth-decomposition figure is not aligned with its textual interpretation; the gap reaches zero before `K/K* = 1`, and the code uses L-regime phi-aware values despite H-regime wording in the paper.
- The appendix overstates implemented computational verification for Proposition 3.
- Overall coverage is only `45%`, with the entire paper-figure layer untested.

### Minor Issues
- `growth_option_decomposition()` double-counts regime-switch value in regime `L`.
- The `regime` parameter in some valuation functions is unused or misleading in practice.
- `paper/generate_figures.py` uses the paper style for both PDF and PNG branches despite the comment claiming separate paper/talk styles.

## Overall Recommendation
**Major revision needed.**

The project has a solid foundation, and the baseline core model appears workable. But the current draft is not ready for submission because several headline quantitative claims are not generated consistently by the implemented code, and the manuscript contains at least one direct contradiction about the main asymmetry result. The best target remains **RFS** once those issues are fixed, because the paper's strongest contribution is a strategic corporate-finance mechanism with a modern, policy-relevant application rather than a theorem-first econometric or pure theory contribution.
